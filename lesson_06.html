Тема пари

В ході пари ми познайомимось з формами не тільки як з набором полів, а і як з важливою частиною зручності користувача. Також розберемо семантику і базовий UX.

1. Основи форми

form, label, input

Кожне поле має бути підписане.

label, пов’язаний через for і id, збільшує клікабельну область і покращує доступність.

Не залишайте поле без зрозумілого підпису.

HTML

<label for="email">Email</label> <input id="email" type="email">

fieldset і legend

Коли поля належать до однієї групи, їх зручно об'єднувати через fieldset.

legend показує загальну назву групи.

2. Стани і UX

focus-visible, hover, disabled

Користувач повинен бачити, яке поле активне.

Не прибирайте outline без заміни — інакше форма стане менш доступною.

CSS

input:focus-visible { outline: 2px solid #2563eb; outline-offset: 2px; }

3. Декоративні ефекти

border-radius, box-shadow, overflow

Такі ефекти не замінюють структуру, але роблять інтерфейс охайнішим.

Їх варто додавати помірно, без перевантаження дизайну.

Практична робота на парі

  1. Створіть форму реєстрації або заявки.
  2. Зв’яжіть усі label з відповідними полями.
  3. Оформіть стани :focus-visible і :hover.
  4. Згрупуйте частину полів через fieldset.
  5. Додайте box-shadow і border-radius.

Домашнє завдання

  1. Створіть форму запису на курс або оформлення замовлення.
  2. Додайте: ім’я, телефон, email, формат навчання, коментар, чекбокс згоди.
  3. Оформіть поля так, щоб стан фокусу був добре помітний.
  4. Зробіть кнопку disabled, якщо хочете показати додатковий стан у дизайні.
  5. У верхній частині сторінки додайте короткий декоративний банер або заголовок.
Практику і домашнє завдання залишено без ускладнення. Основне розширення у цьому файлі — це пояснення та окремі приклади для кожної частини теми.
