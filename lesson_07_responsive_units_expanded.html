Тема пари 7. Responsive Design: Mobile First, viewport, media queries, одиниці вимірювання

Розширена версія з адаптивною версткою, одиницями вимірювання та повними прикладами
Теорія, пояснення, готові приклади, практика і домашня робота

Тема пари

В ході пари ми познайомимось із підходом Mobile First, роботою viewport, медіа-запитами, принципами адаптивної верстки та одиницями вимірювання в CSS: px, %, em, rem, vw, vh, vmin, vmax.

Усі приклади нижче подані як самодостатні фрагменти або як повні HTML-документи, які можна копіювати в окремий файл і запускати без додаткового контексту.

Короткий теоретичний блок

Responsive Design зручніше будувати поступово: від малого екрана до більшого.

Media queries мають з’являтися там, де це потрібно контенту, а не заради самої кількості breakpoints.

Розширений теоретичний блок

Responsive Design — це не просто набір медіа-запитів, а спосіб мислити сторінкою так, щоб вона залишалася читабельною і зручною на різних екранах. Саме тому адаптивність краще закладати від початку, а не додавати в останній момент.

Mobile First як стратегія

Починати з малого екрана зручно, бо це змушує одразу виділити головне: порядок блоків, ширину, читабельність, відступи. Після цього макет легше розширювати через min-width, ніж навпаки — намагатися «згорнути» складну десктопну версію.

viewport як базова умова

Без коректного viewport сторінка на мобільному виглядає неприродно зменшеною. Це одна з тих дрібниць, без яких адаптивний дизайн технічно буде неповним.

Media queries і breakpoints

Медіа-запити варто додавати не заради самої їх наявності, а тоді, коли контент починає виглядати незручно: картки стають надто вузькими, текст стискається, блоки злипаються або виникає зайва порожнеча. Саме тому breakpoint краще вибирати по контенту, а не по назві пристрою.

Адаптивна верстка: що саме має змінюватися

Адаптивна верстка — це підхід, за якого сторінка не просто «зменшується» на маленькому екрані, а перебудовується так, щоб не втрачати зручність. На різних ширинах можуть змінюватися кількість колонок, розмір шрифту, відступи, порядок блоків, ширина зображень, розмір кнопок і навіть спосіб показу меню.

Головна ідея: користувач не повинен збільшувати сторінку пальцями, горизонтально прокручувати текст або шукати важливий блок, який «поїхав» за межі екрана.

Що найчастіше адаптують

  • Сітку сторінки: 1 колонка на телефоні, 2 на планшеті, 3–4 на десктопі.
  • Меню: на великому екрані горизонтальне, на малому — вертикальне або компактне.
  • Картки: на телефоні займають всю ширину, на десктопі стоять у ряд.
  • Зображення: не виходять за межі контейнера, мають max-width: 100%.
  • Типографіку: заголовки і текст не мають бути надто великими або надто дрібними.
  • Відступи: на маленьких екранах їх часто зменшують, щоб не витрачати простір.
Важливо: адаптивність — це не лише @media. Частину адаптивності можна зробити одразу через max-width, width: 100%, flex-wrap, auto-fit, minmax() та відносні одиниці вимірювання.

Одиниці вимірювання в CSS

У CSS є абсолютні та відносні одиниці. Для адаптивної верстки важливо розуміти, коли краще використовувати фіксовані значення, а коли — значення, які залежать від екрана, контейнера або розміру шрифту.

Одиниця Що означає Де зручно використовувати
px Пікселі. Фіксована одиниця. Тонкі рамки, точні іконки, невеликі елементи, border-radius.
% Відсоток від розміру батьківського елемента. Ширина блоків, зображення, колонки.
em Залежить від розміру шрифту поточного елемента. Внутрішні відступи кнопок, елементи, які мають масштабуватися разом із текстом.
rem Залежить від розміру шрифту кореневого елемента html. Загальні відступи, розміри шрифтів, стабільна система розмірів на сайті.
vw 1% від ширини viewport. Адаптивні hero-блоки, великі заголовки, елементи, залежні від ширини екрана.
vh 1% від висоти viewport. Перший екран сайту, секції на всю висоту екрана.
vmin 1% від меншої сторони viewport. Елементи, які мають однаково добре виглядати і в портретній, і в альбомній орієнтації.
vmax 1% від більшої сторони viewport. Великі декоративні елементи, фонові форми, масштабні блоки.

Коротке правило вибору

  • px — коли потрібна точність.
  • % — коли елемент має залежати від ширини батьківського блоку.
  • em — коли відступи мають масштабуватися від розміру тексту самого елемента.
  • rem — коли потрібна єдина система розмірів на всій сторінці.
  • vw і vh — коли елемент має залежати від розміру вікна браузера.

Приклади з одиницями вимірювання

Приклад 6. Відсотки для адаптивної ширини

Блок має займати всю доступну ширину, але не бути надто широким на великому екрані. Для цього часто комбінують width: 100% і max-width.

Цей блок займає всю доступну ширину, але має обмеження max-width.

Повний приклад коду

<!DOCTYPE html> <html lang="uk"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Width and max-width</title> <style> body { margin: 0; font-family: Arial, sans-serif; padding: 24px; } .card { width: 100%; max-width: 620px; padding: 20px; background: #dbeafe; border-radius: 14px; } </style> </head> <body> <div class="card"> Блок адаптується під ширину екрана, але не стає занадто широким. </div> </body> </html>

Приклад 7. rem для стабільної системи відступів

rem зручно використовувати для розмірів, які мають бути однаковими по всій сторінці. Якщо змінити базовий розмір шрифту в html, пропорційно зміняться всі значення в rem.

padding: 1.5rem; border-radius: 1rem;

Повний приклад коду

<!DOCTYPE html> <html lang="uk"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>rem units</title> <style> html { font-size: 16px; } body { font-family: Arial, sans-serif; padding: 2rem; } .card { padding: 1.5rem; border-radius: 1rem; background: #e0f2fe; } </style> </head> <body> <div class="card"> rem допомагає будувати єдину систему розмірів. </div> </body> </html>

Приклад 8. em для кнопки

em залежить від розміру шрифту самого елемента. Тому кнопка з padding в em збільшується разом із текстом.

Повний приклад коду

<!DOCTYPE html> <html lang="uk"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>em button</title> <style> body { font-family: Arial, sans-serif; padding: 30px; } .button { font-size: 20px; padding: 0.8em 1.4em; border: none; border-radius: 0.8em; background: #1d4ed8; color: white; } </style> </head> <body> <button class="button">Записатися</button> </body> </html>

Приклад 9. vh для першого екрана

vh часто використовують для першого екрана сайту. Наприклад, min-height: 100vh дозволяє зробити секцію не меншою за висоту вікна браузера.

У реальному прикладі це може бути min-height: 100vh

Повний приклад коду

<!DOCTYPE html> <html lang="uk"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Hero with vh</title> <style> body { margin: 0; font-family: Arial, sans-serif; } .hero { min-height: 100vh; display: grid; place-items: center; padding: 2rem; background: #dbeafe; text-align: center; } </style> </head> <body> <section class="hero"> <div> <h1>Адаптивний перший екран</h1> <p>Секція займає всю висоту viewport.</p> </div> </section> </body> </html>

Приклад 10. Адаптивний розмір тексту через clamp()

clamp() дозволяє задати мінімальне, бажане і максимальне значення. Це зручно для заголовків, які мають плавно змінюватися залежно від ширини екрана.

Адаптивний заголовок

Повний приклад коду

<!DOCTYPE html> <html lang="uk"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>clamp title</title> <style> body { font-family: Arial, sans-serif; padding: 30px; } h1 { font-size: clamp(28px, 6vw, 64px); } </style> </head> <body> <h1>Заголовок плавно змінює розмір</h1> </body> </html>

Адаптивність без зайвих media queries

Не кожна адаптивна поведінка потребує окремого breakpoint. Часто можна написати гнучкий код, який сам підлаштовується під доступну ширину.

Типові помилки в адаптивній верстці

  • Фіксована ширина блоку, наприклад width: 1200px, через яку на мобільному з’являється горизонтальний скрол.
  • Зображення без max-width: 100%, які виходять за межі контейнера.
  • Занадто багато breakpoints без реальної потреби.
  • Занадто маленький текст на мобільному екрані.
  • Великі відступи на телефоні, які забирають половину екрана.
  • Використання тільки px там, де краще підійшли б %, rem, vw або vh.

Практична робота на парі

  1. Створіть секцію з картками за принципом Mobile First.
  2. Додайте viewport у HTML.
  3. Налаштуйте 1 колонку на мобільному, 2 на планшеті, 4 на десктопі.
  4. Додайте адаптивну ширину через width: 100% і max-width.
  5. Використайте rem для загальних відступів і em для кнопок.
  6. Створіть hero-секцію з min-height: 100vh.
  7. Зробіть один блок адаптивним без media queries через repeat(auto-fit, minmax()).
  8. Поясніть, чому breakpoints краще обирати по контенту.

Домашнє завдання

  1. Створіть адаптивну сторінку для послуг, товарів, курсів або портфоліо.
  2. Додайте header, hero-блок, секцію з картками, секцію з описом і footer.
  3. На мобільному екрані основні блоки мають іти в одну колонку.
  4. На середній ширині картки мають перебудовуватися у дві колонки.
  5. На великій ширині картки мають перебудовуватися у три або чотири колонки.
  6. Використайте мінімум один приклад з rem, em, %, vw або vh.
  7. Додайте адаптивний заголовок через clamp().
  8. Перевірте сторінку в режимі responsive у DevTools.
  9. На сторінці не має бути горизонтального скролу.
